Eviction policy guide
Pilih policy berdasarkan pola akses:
• LRU — data yang baru diakses kemungkinan besar diakses lagi. Default terbaik untuk kebanyakan kasus.
• LFU — data populer harus tetap di cache meski belum diakses baru-baru ini (e.g. trending content).
• FIFO — sederhana, cocok untuk batch processing sequential.
• TTL — data punya masa berlaku alami: session, token, API response.

Redis maxmemory-policy options:
noeviction · allkeys-lru · volatile-lru · allkeys-lfu · volatile-ttl · allkeys-random
